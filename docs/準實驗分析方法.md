# 準實驗量化分析：架構設計與 SPSS 對齊說明（草案，待確認）

本文件規劃一個**與序列分析完全獨立**的量化分析模組，用於準實驗設計
（前測／後測、實驗組／對照組、不同向度題目）的統計檢定，並要求**數學公式
100% 對齊 SPSS**。實作前請先確認文末的「SPSS 對齊注意事項（待確認）」。

---

## 一、目的與範圍

準實驗設計常見分析：

- 前測 vs 後測（同一組前後差異）
- 實驗組 vs 對照組（組間差異）
- 依題目**向度**（例如深入題／簡單題、不同認知向度）分別比較，找出哪些向度有顯著差異、哪些沒有
- 每個結果都要附**描述統計 + 效果量 + 中文解釋**

涵蓋的檢定：描述統計、獨立樣本 t 檢定、成對樣本 t 檢定、單因子 ANOVA、
**ANCOVA（共變數分析）**、線性迴歸。

---

## 二、架構與獨立性（不與序列分析打架）

```
src/quant/                    ← 全新獨立套件，不 import src/sequence 或 src/core
├── __init__.py
├── loader.py                 ← 讀取 data/量化/ 的前後測資料
├── cleaning.py               ← 資料清洗（缺值、異常、型別、向度標記檢查）
├── stats.py                  ← SPSS 對齊的統計檢定（純函式，可測試）
└── report.py                 ← 結果彙整與中文解釋

pages/7_📈_準實驗分析.py       ← 唯一前端入口（分頁：描述統計 / t 檢定 / ANOVA / ANCOVA / 迴歸）
data/量化/                     ← 量化資料（gitignore，不上傳）
tests/test_quant.py           ← 以手算值與 SPSS 已知結果交叉驗證
docs/準實驗分析方法.md          ← 本文件
```

**隔離原則**：`src/quant` 不依賴序列分析任何程式；共用的只有 Streamlit 入口與
session state。改動量化模組不影響序列分析，反之亦然。

## 三、資料格式與清洗

### 預期欄位（寬格式，一列一位學生）

| 欄位 | 說明 |
| --- | --- |
| 學生ID | 學生識別碼 |
| 組別 | 實驗組 / 對照組 |
| 前測_總分（或各題） | 前測成績 |
| 後測_總分（或各題） | 後測成績 |
| 各題向度標記 | 標示每題屬於哪個向度（深入／簡單／…） |

> 前測全班相同、後測部分題目屬不同向度 → 依向度把後測分數彙整，各向度分開檢定。
> **向度如何在你的資料中標記，待你把量化資料放進 `data/量化/` 後再一起確認欄位對應。**

### 清洗步驟

1. 去除完全空白列、重複列。
2. 型別轉換：分數轉數值，無法轉換者標記為缺值。
3. 缺值處理：**listwise deletion（整列刪除）**，與 SPSS 預設一致。
4. 異常值：先只「標記與報告」，不自動刪除（是否處理由你決定）。
5. 一致性檢查：組別值域、前後測題數、向度標記是否齊全。

## 四、統計方法與 SPSS 對齊

| 分析 | 統計量 | 效果量 | SPSS 對應 |
| --- | --- | --- | --- |
| 描述統計 | n、平均、SD、SE、最小/最大、偏態、峰度 | — | Descriptives / Explore |
| 獨立樣本 t | t、df、雙尾 p、95% CI；含 Levene | Cohen's d（pooled） | Independent-Samples T Test |
| 成對樣本 t | t、df、雙尾 p、95% CI | Cohen's d_z（差異分數 SD） | Paired-Samples T Test |
| 單因子 ANOVA | F、df、p；含 Levene、（可選 Welch） | partial η²、（可選 ω²） | One-Way ANOVA / GLM |
| ANCOVA | F、df、p（Type III SS）、調整後平均 | partial η² | GLM Univariate |
| 迴歸 | b、標準化 β、t、p、R²、調整後 R²、F | — | Linear Regression |

**建議套件**：`pingouin`（專為貼近 SPSS 輸出而設計，提供 Cohen's d、partial η²、
ANCOVA、post-hoc 校正）搭配 `statsmodels`（Type III ANCOVA / 迴歸）。純 scipy 在
效果量、Levene 中心、post-hoc 等處與 SPSS 預設不同，需自行對齊。

## 五、SPSS 對齊注意事項（★ 待你逐項確認）

以下為 Python 與 SPSS 可能不一致之處，我的預設處理法均以「對齊 SPSS」為原則：

1. **統計套件**：擬加入 `pingouin` + `statsmodels` 依賴（最貼近 SPSS）。是否同意？
2. **Levene 檢定中心**：SPSS 預設用**平均數**；scipy 預設用中位數。→ 我固定用 `center="mean"`。
3. **獨立樣本 t**：SPSS 同時輸出「假設等變異(pooled)」與「不假設等變異(Welch)」兩列。→ 兩列都報，並依 Levene 給建議看哪一列。
4. **Cohen's d（獨立樣本）**：分母用 **pooled SD**（SPSS 作法）。是否**同時**附 Hedges' g（小樣本校正）？
5. **成對樣本 d**：SPSS 用**差異分數的 SD**（d_z）當分母。確認採此定義（與部分文獻的平均 SD 不同）。
6. **ANOVA 效果量**：預設報 **partial η²**（SPSS GLM）。是否**加報** ω²？
7. **ANOVA post-hoc**：預設 **Tukey HSD**；變異數不同質時可改 **Games-Howell**。是否需要、用哪種？
8. **ANCOVA 設計**：前測=共變量、組別=因子、後測=依變量；**Type III 平方和**（SPSS 預設）；
   先檢定**迴歸斜率同質性**（組×共變量交互作用不顯著才適用）；輸出**調整後平均數**與 partial η²。確認此設計。
9. **缺值**：**listwise deletion**（SPSS 預設）。
10. **信賴區間**：預設 **95% 雙尾**。
11. **標準化迴歸係數 β**：SPSS 迴歸會輸出，我一併計算。
12. **顯示精度**：報表顯示到小數 3 位（計算用全精度），與 SPSS 慣例一致。
13. **向度分類欄位**：後測依向度分組跑檢定 → 需你確認向度在資料中的標記欄位（等你放資料）。
