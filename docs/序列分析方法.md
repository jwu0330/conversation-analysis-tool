# Bloom 提問序列分析：方法與公式說明

本文件說明本工具「序列分析」的完整計算邏輯與公式，並保證：
**網頁 App 與獨立命令列腳本（CLI）算出的結果完全一致**，可供重複驗證。

---

## 一、一致性保證（最重要）

所有計算集中在單一份核心程式：

```
src/sequence/analysis.py  →  函式 analyze()
```

- 網頁 App（`pages/6_🔀_序列分析.py`）呼叫 `analyze()`。
- 獨立 CLI（`scripts/run_sequence_analysis.py`）也呼叫**同一個** `analyze()`。

因為兩者共用同一份程式碼與同一組參數，對相同輸入必然得到相同輸出。
這件事由測試 `tests/test_gseq.py::test_app_and_cli_are_identical` 自動驗證
（用 `pandas.testing.assert_frame_equal` 逐格比對轉移表、GSEQ 表、序列表與轉移矩陣）。

> 換言之，「一模一樣」不是靠人工比對，而是**架構上保證 + 自動化測試釘死**。

## 二、分析單位

| 層級 | 定義 |
| --- | --- |
| 事件 | 一題學生提問 |
| 事件編碼 | 該題的 Bloom Level（L0–L6） |
| 序列 | 一位學生依時間排序的所有提問 |
| 比較單位 | 組別（實驗組 / 對照組） |

Lag = 1（只看相鄰兩題的轉移）。

---

## 三、計算流程（對應函式）

| 步驟 | 說明 | 函式 |
| --- | --- | --- |
| 1. 清理 | 取出四欄、解析 Bloom 數字、去除缺漏、可選排除 L0 | `prepare()` |
| 2. 個人序列 | 每位學生依排序欄位排成 Bloom 序列 | `build_sequences()` |
| 3. Lag-1 轉移 | 每位學生相鄰兩題 (Source→Target)，依組別彙整次數 | `transitions()` |
| 4. 轉移矩陣 | 攤成 Source×Target 方陣（次數或列機率） | `transition_matrix()` |
| 5. GSEQ 統計 | 期望次數、轉移機率、調整殘差 z、p 值、顯著 | `gseq_stats()` |
| 6. 高低階 | 併成高/低階後的四種轉移 | `high_low_transitions()` |
| 7. 題序剖面 | 各組第 1、2、3… 題的平均 Bloom | `position_profile()` |

排序穩定性：以 `(學生, 排序欄位)` 做 **stable sort**，同鍵時保留原始列順序。

## 四、GSEQ 式滯後序列分析公式

令轉移次數矩陣 **O**（列 = 前題 Source *i*，欄 = 後題 Target *j*）：

- **N** = 總轉移數 = ΣΣ O_ij
- **R_i** = 第 *i* 列總和（從 *i* 出發的轉移數）
- **C_j** = 第 *j* 欄總和（進入 *j* 的轉移數）

### 1. 期望次數（在「前後題互相獨立」假設下）

```
E_ij = R_i × C_j / N
```

### 2. 轉移機率（條件機率）

```
P_ij = O_ij / R_i  = P(下一題 = j | 前一題 = i)
```

### 3. 調整後殘差 z（Adjusted Residual）

採 Allison & Liker (1982)、Bakeman & Gottman (1997) 的公式：

```
z_ij = (O_ij − E_ij) / sqrt( E_ij × (1 − R_i/N) × (1 − C_j/N) )
```

### 4. 雙尾 p 值

```
p_ij = 2 × (1 − Φ(|z_ij|))        Φ 為標準常態累積分布
```

### 5. 顯著判準

| 條件 | 判定 |
| --- | --- |
| z > z_crit | ↑ 顯著偏多（此轉移出現得比隨機預期多） |
| z < −z_crit | ↓ 顯著偏少 |
| 其餘 | 未達顯著 |

α = 0.05 時 z_crit ≈ 1.96。

### 手算驗證範例（見測試）

O = [[1, 3], [2, 4]]（L1/L2），N = 10，R₁ = 4，C₁ = 3：

- E₁₁ = 4 × 3 / 10 = **1.2**
- P₁₁ = 1 / 4 = **0.25**
- z₁₁ = (1 − 1.2) / √(1.2 × 0.6 × 0.7) = −0.2 / √0.504 ≈ **−0.28**

此範例由 `tests/test_gseq.py::test_gseq_formula_by_hand` 驗證。

## 五、獨立 CLI 用法

不需開網頁，可直接在終端機重現全部結果：

```bash
python scripts/run_sequence_analysis.py sample_data/conversation_sample.xlsx \
    --student 學生ID --group 組別 --bloom Bloom層級 --order 對話ID \
    --high-min 4 --alpha 0.05 --output 序列分析結果.xlsx
```

- 欄位參數可省略（自動辨識）。
- `--exclude-l0` 排除 L0；`--output` 輸出含所有結果的 Excel。
- 螢幕會列出轉移表與各組**達顯著**的轉移。

## 六、與正式 GSEQ 軟體的關係與差異

| 項目 | 本工具 | GSEQ 軟體 |
| --- | --- | --- |
| 期望次數、調整殘差 z、p 值 | ✅ 有（同公式） | ✅ 有 |
| 轉移機率、顯著判定 | ✅ 有 | ✅ 有 |
| 多重 Lag（Lag 2、3…） | ✗ 目前僅 Lag-1 | ✅ |
| Yule's Q、Kappa 等其他統計 | ✗（可再擴充） | ✅ |
| 資料格式轉檔（MDS/SDS） | ✗ | ✅ |

**結論**：本工具在 **Lag-1 的核心指標（期望次數、調整殘差 z、顯著轉移）** 上與 GSEQ
使用相同公式，可用於初步探索與簡報；若需多重 Lag 或其他進階統計，再把本工具整理好的
序列/轉移資料匯出，轉入正式 GSEQ 進一步分析。

---

## 七、假設與限制

1. Lag-1、所有轉移皆允許（未設結構性零格）。
2. 調整殘差以常態近似檢定，**每格期望次數過低（< 5）時 z 的近似會變差**，需謹慎解讀。
3. 自我迴圈（如 L4→L4，連續同層級）計入分析。
4. 排序正確性取決於「排序依據」欄位；建議使用單調遞增的題序或時間戳。
