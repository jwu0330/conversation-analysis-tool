## 四、📐 統計分析：SPSS 式 ANCOVA（`pages/2_📐_統計分析.py` + `core/ancova.py`）

### 操作流程
- **預設路徑**：依變數＝後測、共變量＝前測、組別＝實驗/對照或班級（依欄名自動猜測）。
- 按 **「▶️ 一鍵執行 ANCOVA」** 就跑完整套 SPSS 流程。
- **進階設定**（收合）可改依變數／組別／共變量（可多共變量）／顯著水準 α；不展開就用預設。

### 一鍵會依序輸出的 6 張表（對照 SPSS「一般線性模型 → 單變量」）

| 步驟 | 輸出 | 統計方法 |
| --- | --- | --- |
| ① 描述統計 | 各組 n / 平均 / 標準差 | `groupby` |
| ② 變異數同質性 | Levene W、p、是否同質 | `scipy.stats.levene(center="mean")` |
| ③ **迴歸斜率同質性** | 組別×共變量交互作用 F、p（ANCOVA 最關鍵前提） | `Y ~ C(G) + X + C(G):X` 的型 III `anova_lm` 交互作用列 |
| ④ 主分析 ANCOVA 表 | 型 III 平方和、F、p、**淨 η²** | `Y ~ C(G) + X` 的 `anova_lm(typ=3)` |
| ⑤ 調整後平均數 EMMeans | 共變量固定在總平均下各組的調整後平均、SE、CI | 模型 `get_prediction`（covar＝grand mean） |
| ⑥ 事後兩兩比較 | 調整後均差、t、p（Bonferroni 校正） | `OLS.t_test` 對比向量（用完整共變異數矩陣） |

### 關鍵公式
- 淨 η²（partial eta²）＝ `SS_effect / (SS_effect + SS_residual)`
- 型 III 平方和＝各效果在「其他效果都已進入模型」下的邊際貢獻（SPSS 預設；對應題主要的「選第三類」）。
- 調整後平均數：以共同斜率模型 `ŷ = a + Σ bₖ·(covar=總平均) + 組別效果` 代入預測。
- 兩兩差異只跟組別項有關（共變量在總平均相減時抵消），SE 取自模型 t\_test，等同 SPSS EMMeans 對比。

### 已驗證
以合成資料（n=60，實驗組真實效果 +8）跑出：調整後平均差 ≈ 8.02、共變量斜率 ≈ 0.6，數值正確。

### 待你確認 / 後續驗證
- 事後比較目前用 **Bonferroni**；SPSS 預設常用 **LSD / Sidak / Tukey**，要哪一種可調。
- 斜率不同質（③顯著）時目前只警示不擋；是否要自動改走 Johnson–Neyman 待定。

## 五、🔀 序列分析（`pages/3_🔀_序列分析.py` + `sequence/analysis.py`）

7 分頁 → **3 區塊**：

### ① 個人序列
每位學生一列的 Bloom 提問時間序列（`build_sequences`）。

### ② 轉移分析
- **轉移表**：Lag-1 相鄰兩題 Source→Target 次數（`transitions`）。
- **轉移圖與矩陣**：六邊形圈箭頭圖 + Sankey + 轉移矩陣熱圖（次數/列機率）。
- **GSEQ 顯著轉移**（滯後序列分析）——本次重點修正：
  - 調整後殘差 z＝`(O−E)/√(E(1−Rᵢ/N)(1−Cⱼ/N))`，`|z|>1.96` 為顯著。
  - **預設只顯示「顯著偏多(↑)」**（正殘差＝研究要找的真實模式）。
  - **「顯著偏少(↓)」預設收起**，需勾選才顯示，並註明「一般不下結論」。
  - **期望次數 < 5** 的格子 z 不可靠，`gseq_stats` 新增 `可信度` 欄標「低(期望<5)」。
- **高低階轉移**（低→高／高→高／高→低／低→低）移入收合區。

### ③ 趨勢分析
- **題序流動**：各組每一題平均 Bloom（`position_profile`）。
- **趨勢／軌跡**：題序×Bloom 線性迴歸（斜率/R²/p）、95% 信賴區帶、個別學生斜率箱型圖。

### 為什麼這樣改（老師說「實驗不正確」的點）
1. 舊版把「顯著偏多」與「顯著偏少」平等呈現，容易誤導——已改成偏多為主、偏少選配。
2. 舊版低期望次數照標紅——已加 `可信度` 欄提醒 z 不可靠。
3. 舊版 7 分頁有重複視角（轉移表／矩陣／GSEQ 是同一份資料三種樣子）——已合併成「轉移分析」一區。

### 待你確認
- 高低階的「高階起始 Level」預設 L4，是否符合你的 Bloom 分層定義。
- GSEQ 是否只需 Lag-1（目前）或要多重 Lag。

## 六、資料流與匯出

所有界面的結果表都呼叫 `state.register_export_table(name, df)` 累積到 session，
使用者在**首頁→匯出**分頁一次輸出成 Excel（每張表一個工作表，另附「分析參數」表）。
`state.py` 為唯一的跨頁狀態層（目前扮演 BFF/中介的角色）。
