## 二、首頁（`app.py`）

兩個分頁：

1. **資料上傳與檢查**：上傳 xlsx/xls/csv 或載入內建資料 → 顯示筆數/欄位/缺漏/重複四個指標 →
   資料預覽 → 欄位型態辨識 → 缺漏值/重複明細（收合）。上傳新檔會清空舊的匯出清單。
2. **匯出分析結果**：各界面用 `state.register_export_table()` 累積的表格集中在此，
   連同分析參數一次輸出成單一 Excel。

## 四、📐 統計分析：SPSS 式 ANCOVA（`pages/2_📐_統計分析.py` + `core/ancova.py`）

> **完全對齊**臺師大吳書豪《量化統計方法實作》SPSS 講義（專案上層 `量化統計方法實作 2025-01-11A.pdf`）。
> 版面分「**重要區**」（依講義順序、必報項目）與「**補充區**」（本工具加值）。

### 操作流程
- **預設路徑**：依變數＝後測、共變量＝前測、組別＝實驗/對照或班級（依欄名自動猜測）。
- 按 **「▶️ 一鍵執行 ANCOVA」** 依講義流程一次跑完；**進階設定**可改變數與 α（可多共變量）。
- 講義提醒（p30）：前測**不要**單獨跑 t 檢定證明同質，直接以前測作共變量。

### 重要區：依講義順序輸出（每項對齊講義）

| 步驟 | 輸出 | 方法／對應講義 |
| --- | --- | --- |
| ① 描述統計 | 各組 N / 平均數 / 標準差 | 講義 p27 |
| ② 常態性檢定 | 各組 Shapiro-Wilk（n<50）或 K-S/Lilliefors（n≥50） | 講義 p21 假設一 |
| ③ 變異數同質性 | Levene **F、df1、df2、p** | `scipy.stats.levene(center="mean")`，p28 |
| ④ 組內迴歸係數同質性 | 組別×前測**交乘項（放最後）** F、p，型 III | `Y ~ C(G,Sum)+X+C(G,Sum):X`，p24-25 |
| ⑤ 主分析 ANCOVA | **主旨間效果檢定完整表**（修正的模型／截距／共變量／組別／錯誤／總計／校正後總數＋型III平方和、df、MS、F、p、**局部 η²**、R²） | 完全因子設計、型 III，p26-27 |
| ⑥ 調整後平均數 | EMMeans：調整後平均、**標準誤**、95% CI | `get_prediction`（covar＝總平均），p27 |
| ⑦ APA 彙整表 | `Group｜N｜Mean｜SD｜Adjusted Mean｜Adjusted SD｜F｜η²` | 講義 p27/p30 報告格式 |

### 補充區
- η² 判準（Cohen 1988：.01/.059/.138）
- 事後兩兩比較（**LSD**，對應講義選項「信賴區間調整＝LSD」）
- **文章寫法**（中/英，套講義 p28-29 模板自動填數字）
- **假設違反處理建議**（p48）：常態不滿足→無母數 U；變異數不同質→前後測跑 t；斜率不同質→詹森-內曼法

### 關鍵公式（對齊 SPSS 表定義）
- 局部 η²（partial eta²）＝ `SS_effect / (SS_effect + SS_error)`
- 修正的模型 SS ＝ 校正後總數 − 錯誤；校正後總數 ＝ Σ(y−ȳ)²；總計 ＝ Σy²
- 型 III 平方和以 sum-to-zero 對比計算，對齊 SPSS/SAS「類型 III」。

### 已驗證
合成資料（n=44）結構恆等式成立：**校正後總數 = 修正的模型 + 錯誤**、**總計 = 校正後總數 + n·mean²**，
確認整張表與 SPSS 定義一致。待提供真實資料再逐格核對數值。

## 五、🔀 序列分析（`pages/3_🔀_序列分析.py` + `sequence/analysis.py`）

7 分頁 → **3 區塊**：

### ① 個人序列
每位學生一列的提問節點時間序列（`build_sequences`）。節點＝知識點、知識點類別或認知層級（SOLO/Bloom），由「節點欄位」決定；可另選「顯示名稱欄位」讓圖上顯示文字（如「檔案傳輸類」）而非代碼。

### ② 轉移分析
- **轉移表**：Lag-1 相鄰兩題 Source→Target 次數（`transitions`）。
- **轉移圖與矩陣**：六邊形圈箭頭圖 + Sankey + 轉移矩陣熱圖（次數/列機率）。
- **GSEQ 顯著轉移**（滯後序列分析）——本次重點修正：
  - 調整後殘差 z＝`(O−E)/√(E(1−Rᵢ/N)(1−Cⱼ/N))`，`|z|>1.96` 為顯著。
  - **預設只顯示「顯著偏多(↑)」**（正殘差＝研究要找的真實模式）。
  - **「顯著偏少(↓)」預設收起**，需勾選才顯示，並註明「一般不下結論」。
  - **期望次數 < 5** 的格子 z 不可靠，`gseq_stats` 新增 `可信度` 欄標「低(期望<5)」。
- **高低階轉移**（低→高／高→高／高→低／低→低）移入收合區。

### ③ 趨勢分析
- **題序流動**：各組每一題平均節點碼（`position_profile`；僅對有序的層級型節點有意義）。
- **趨勢／軌跡**：題序×節點碼 線性迴歸（斜率/R²/p）、95% 信賴區帶、個別學生斜率箱型圖（同上，僅層級型節點有意義）。

### 為什麼這樣改（老師說「實驗不正確」的點）
1. 舊版把「顯著偏多」與「顯著偏少」平等呈現，容易誤導——已改成偏多為主、偏少選配。
2. 舊版低期望次數照標紅——已加 `可信度` 欄提醒 z 不可靠。
3. 舊版 7 分頁有重複視角（轉移表／矩陣／GSEQ 是同一份資料三種樣子）——已合併成「轉移分析」一區。

### 待你確認
- 高低階的「高階起始碼」預設 4，僅在節點為「有序層級」（如 SOLO/Bloom）時才有意義；知識點／類別是名目型，高低階與趨勢分析請略過、以轉移分析為主。
- GSEQ 是否只需 Lag-1（目前）或要多重 Lag。

## 六、資料流與匯出

所有界面的結果表都呼叫 `state.register_export_table(name, df)` 累積到 session，
使用者在**首頁→匯出**分頁一次輸出成 Excel（每張表一個工作表，另附「分析參數」表）。
`state.py` 為唯一的跨頁狀態層（目前扮演 BFF/中介的角色）。
